# CHAT.md

## Role

A default Senter canvas building block for the live conversation.

## Display

Show the current conversation and allow context blocks to be opened beside it. Keep the STS auxiliary path fast and tool-free.

## GOOP

Conversation evidence feeds Context reconciliation. Repeated requests become Skills; coherent clusters may become Agents; stable workflows may become proposed Loops.

## Safety

The chat block does not grant tools or permissions by itself.

Success is measured by response latency, continuity, and correct topic/context attachment.

The user remains the authority for applying proposals.

The user may edit this block as Markdown source.

The block must remain readable in the terminal and Android canvas.

The block should degrade gracefully if the gateway is unavailable.

The block should never include credentials.

The block may link to source records.

The block can be duplicated for experiments.

The block's state is not executable code.

The block is part of the Senter Agent language layer.

The block may later gain morphable UI metadata.

The block is complete.
